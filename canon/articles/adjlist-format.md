# AdjList Format

> **What this answers.** What's an AdjList? When does the agent need to write one? What's the file format and naming convention?
>
> **Related articles.** `klappy://canon/articles/payload-construction` · `klappy://canon/articles/file-system-map`

---

## What an AdjList is

In the GUI, paragraph adjustment is a right-click affordance on the preview pane — the user shrinks or grows a specific paragraph by N lines to fix overfull boxes, widows, or orphans. In headless, the same fine-tuning is expressed as a per-book text file: an AdjList.

The agent constructs the AdjList content as text and includes it in the payload's `config_files` map. PTXprint reads it during typesetting and applies the adjustments.

## File location and name

```
shared/ptxprint/<config>/AdjLists/<num><BOOK><prj>-<config>[.SFM|-diglot].adj
```

Composition:

```
<num><BOOK><prj>-<config>[.SFM|-diglot].adj
  │     │     │       │       │
  │     │     │       │       └─ `.SFM.adj` for monoglot, `-diglot.adj` for diglot
  │     │     │       └────────  config name (case as named)
  │     │     └────────────────  project ID (case as named)
  │     └──────────────────────  three-letter USFM book code, uppercase
  └────────────────────────────  two-digit canonical USFM book number, zero-padded
```

Examples:

| Filename | Meaning |
|---|---|
| `41MATWSG-Default.SFM.adj` | Matthew, project WSG, config Default, monoglot |
| `46ROMWSG-FancyNT.SFM.adj` | Romans, project WSG, config FancyNT, monoglot |
| `41MATWSG-DiglotEN-FR-diglot.adj` | Matthew, diglot version |
| `01GENWSG-Default.SFM.adj` | Genesis (book 01), project WSG, config Default |

The book number must be zero-padded to two digits. Use `01`–`66` for canonical books, plus the standard USFM peripheral numbers for non-canonical books.

One file per book. A project that adjusts paragraphs across MAT, ROM, and PSA has three files: `41MATWSG-Default.SFM.adj`, `46ROMWSG-Default.SFM.adj`, `19PSAWSG-Default.SFM.adj`.

## Line syntax

The basic line shape:

```
BKK c.v ±N[P] [% hints | mrk=<marker> expand=<pct>]
```

| Token | Meaning |
|---|---|
| `BKK` | three-letter USFM book code; append `L` or `R` for diglot left/right column |
| `c.v` | chapter.verse reference (e.g. `3.16`); for glossary, keyword reference (e.g. `k.Adam`) |
| `±N` | integer line adjustment; negative shrinks, positive grows; `+0` means no adjustment (hint-only line) |
| `[P]` | optional paragraph index when the verse contains multiple paragraphs |
| `% hints` | candidate paragraphs as `+[2] +[3]`; PTXprint generates these automatically when interactive |
| `mrk=<m>` | extended format: USFM marker whose paragraph text should be scaled |
| `expand=<pct>` | extended format: text scale percentage (`98` = 98% of normal) |

Lines starting with `%` are comments. Blank lines are ignored.

## Worked examples

### Basic shrink/grow

```
% Shrink Romans 2:6 paragraph by one line; grow Matthew 1:1 by one
ROM 2.6 -1
MAT 1.1 +1
```

### With paragraph index

```
% At Matthew 1:1, grow the 4th paragraph at that verse by one line
MAT 1.1 +1[4]
```

Use the `[P]` index when the verse anchors multiple paragraphs and the agent needs to target a specific one.

### Hint-only (auto-generated by PTXprint)

```
% Current adjustment is +0; paragraphs 2 and 3 are candidates if the user wants
% to grow at this reference
MAT 1.1 +0% +[2] +[3]
```

PTXprint's interactive flow generates these when the user hovers over an overfull region. The agent typically doesn't write these directly; if it does, the `+0` says "no actual adjustment yet, just recording candidates."

### Extended: scale paragraph text

```
% Shrink Isaiah 6:11 paragraph by 1 line AND scale its \q-marked text to 98%
ISA 6.11 -1 % mrk=q expand=98

% Grow Isaiah 7:23 paragraph by 1 line AND scale its \p-marked text to 103%
ISA 7.23 +1 % mrk=p expand=103
```

The extended format lets the agent (or user) bias both line count and horizontal scale. Useful when a single line adjustment alone produces awkward spacing.

### Diglot: target one column

```
% In a diglot, MRKL targets the left column at Mark 1:1; MRKR targets right
MRKL 1.1 +0
MRKR 1.1 +2
```

Diglot AdjList filenames end in `-diglot.adj`; the L/R book-code suffixes inside select column.

### Glossary: keyword reference

```
% In the Glossary book (right column), the entry for keyword "Adam" gets adj 0
GLOR k.Adam 0
```

`GLO` is the glossary book code; `R` selects the right column; `k.Adam` is the keyword anchor (no chapter/verse).

## When the agent should write an AdjList

- The user reports an overfull box, widow, or orphan at a specific reference and wants it fixed.
- The user wants to shrink or expand a specific paragraph by N lines without changing global layout.
- The user is in headless context and cannot use the right-click GUI workflow.
- A previous typeset run reported overfull boxes the agent's diagnostic pinned to specific verses.

## When NOT to write an AdjList

- **The user is in the GUI.** Direct them to right-click on the preview pane.
- **The fix is layout-wide.** Adjust paragraph or font settings in `ptxprint.cfg`, not individual paragraphs.
- **The agent is guessing about direction.** Adjustments that go the wrong way make layout worse. If unsure whether a paragraph should shrink or grow, ask the user or make a `mode: "simple"` test run with no adjustment first to see the current state.

## The right magnitude

Paragraph adjustments are usually `±1` or `±2`. Larger values are unusual and may signal the wrong fix:

| Magnitude | Reasonable use |
|---|---|
| `±1` | Most common; one-line tweak |
| `±2` | Occasional; fixing widow/orphan plus line count |
| `±3` or more | Rare; usually means the underlying layout is wrong |

If the agent finds itself proposing `±5`, step back and consider whether the cfg (font size, line spacing, column width) needs adjustment instead.

## Including AdjLists in the payload

```json
"config_files": {
  "shared/ptxprint/Default/ptxprint.cfg": "...",
  "shared/ptxprint/Default/AdjLists/41MATWSG-Default.SFM.adj": "MAT 1.1 +1\nMAT 5.3 -1\n",
  "shared/ptxprint/Default/AdjLists/46ROMWSG-Default.SFM.adj": "ROM 2.6 -1\n"
}
```

One key per book that has adjustments. Books with no adjustments don't need keys.

## Reading existing AdjLists from project state

Before constructing a payload, the agent should read existing AdjLists from project state and include them unchanged in the payload (unless the user is editing them). Existing adjustments matter for layout consistency across runs.

If the user adds new adjustments, the agent merges them into the existing content rather than replacing.
